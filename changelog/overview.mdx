<Update label="February 20, 2026" description="Evaluations: Confidence in Every Conversation">
    Evaluations give you a systematic way to verify that your agent handles real-world scenarios the way you expect. Define a user persona and a set of success criteria, and Dimedove simulates a full multi-turn conversation with your agent, then scores the result with an independent AI judge. Every time you update your agent's instructions, knowledge base, tone, or tasks, you can run your evaluations to confirm nothing broke and everything still works.

    Each evaluation follows a three-step lifecycle: define, run, review. The simulated user stays in character throughout the conversation, writing naturally and reacting realistically to your agent's responses. Once the conversation completes, an independent judge evaluates the full transcript against each criterion you defined, producing a score and a pass, partial, or fail verdict. A detailed reasoning section and per-criterion breakdown show exactly what worked, what did not, and why.

    Run evaluations individually or all at once after making changes. Every run is saved with its score, verdict, and full conversation transcript, so you can track how your agent's performance evolves over time. Pair evaluations with Versions to test a draft configuration before publishing it to production.

    - **Scenario-based testing** with realistic, natural user simulation
    - **AI-powered scoring** with an independent judge evaluating each success criterion
    - **Pass, partial, and fail verdicts** with scoring for granular insight
    - **Full conversation transcripts** for every run, viewable turn by turn
    - **Run individually or all at once** to validate your agent after any change
    - **Run history** to track performance over time and catch regressions

    ### Features

    - Create evaluations with custom user scenarios describing who the visitor is and what they are trying to accomplish
    - Define success criteria that tell the judge exactly what a passing conversation looks like
    - Configure the maximum number of conversation turns per evaluation
    - Simulated users stay in character, write naturally, and react realistically to your agent's responses
    - Independent AI judge evaluates the full transcript against each defined criterion
    - Scoring system with pass, partial, and fail verdicts based on how well your agent met the defined criteria
    - Detailed evaluation reasoning citing specific moments in the conversation
    - Per-criterion breakdown showing which criteria passed or failed and why
    - Full conversation transcript viewable turn by turn for every completed run
    - Run a single evaluation or all active evaluations at once from the dashboard
    - Run history with status, date, score, and verdict for every past execution
    - Visual status indicators for quick scanning: green for pass, yellow for partial, red for fail
    - Evaluations run against your current agent configuration without affecting production conversations, inbox, or contacts
    - Bilingual evaluation output in English and French

    ### Notes

    - Evaluations are available to teams on the Pro subscription

</Update>

<Update label="February 11, 2026" description="Workflows: Automate Across Every Channel">
    Workflows bring event-driven automation to Dimedove. Connect any platform event to a sequence of actions, without writing code or leaving the dashboard. When a form is submitted, an email arrives, a call comes in, an agent qualifies a lead, or a Slack message is received, workflows let you automatically respond with emails, SMS messages, outbound calls, Slack messages, or HTTP requests to external systems.

    Each workflow starts with a trigger (the event that kicks things off) and executes up to two action steps in sequence. Nine trigger types cover every channel on the platform, and optional filters let you narrow triggers to specific agents, forms, phone numbers, email channels, or Slack agents. Six action types give you full cross-channel reach, including a Delay action for timed sequences like waiting before a follow-up call. When actions are sent through agent-assigned channels, they automatically create conversations in your Inbox, so your team has full visibility and agents can continue the interaction.

    Template variables make workflows dynamic: reference any field from the trigger payload in your action configurations using `{{trigger.body.field_name}}` syntax, with dot notation and array index support for nested data. A built-in run history with step-by-step details and an analytics dashboard give you complete visibility into every execution.

    - **Multiple trigger types** covering webhooks, manual runs, qualified opportunities, human assistance requests, form submissions, inbound emails, inbound calls, inbound SMS, and Slack messages
    - **Multiple action types** including HTTP Request, Email, SMS, Call, Slack Message, and Delay
    - **Optional trigger filters** to narrow events to specific agents, forms, phone numbers, email channels, or Slack agents
    - **Template variables** for dynamic data in action configurations
    - **Inbox integration** with automatic conversation creation from email, SMS, and call actions
    - **Run history and analytics** with step-by-step execution details, success rate, and duration tracking

    ### Features

    - Create workflows from the dashboard with a visual editor showing triggers, actions, and connections
    - Choose from multiple trigger types: Webhook, Manual, Qualified Opportunity, Human Required, Form Submission, Email Received, Inbound Call, Inbound SMS, and Slack Message Received
    - Apply optional filters to narrow triggers to specific platform resources
    - Webhook triggers generate a unique URL for receiving external HTTP POST requests
    - Manual triggers add a Run button to the workflow for on-demand execution
    - Configure multiple action nodes per workflow, with Delay supported for timed sequences
    - HTTP Request action supports GET, POST, PUT, PATCH, DELETE with configurable headers, query parameters, and JSON body
    - Email action with three sending modes: Default (Dimedove), Custom sender name, or through a configured Email Channel
    - SMS action from a Dimedove number or a team phone number with agent-assigned inbox conversations
    - Call action initiates outbound voice calls through team phone numbers with voice-enabled agents
    - Slack Message action posts to any channel in your connected workspace
    - Delay action pauses execution for a configurable duration in seconds, minutes, or hours
    - Template variables (`{{trigger.body.field}}`) in all action text fields with dot notation and array index support
    - Three workflow statuses: Draft, Live, and Paused
    - Run history table with status, trigger type, duration, and step progress for every execution
    - Detailed run view showing trigger data, step inputs, outputs, and error messages
    - Analytics dashboard with runs over time, total runs, success rate, and average duration

    ### Notes

    - Workflows are available to teams on the Pro subscription

</Update>

<Update label="January 6, 2026" description="MCP Support: Extend and Expose Your Agents">
    We're introducing full Model Context Protocol (MCP) support, giving your agents two powerful new capabilities: connect to external MCP servers to extend what your agents can do, and deploy your own hosted MCP servers to expose your agents to the broader AI ecosystem. Whether you want to bring third-party tools into your agent conversations or make your agents accessible from ChatGPT, automation platforms, and other AI tools, MCP makes it possible.

    MCP Tasks let your agents connect to any MCP-compatible server during conversations. Configure the server URL, choose your transport protocol, add custom headers for authentication, and your agent automatically discovers and uses the available tools. This opens up integrations with specialized services, custom APIs, and external capabilities that extend your agent far beyond its built-in functionality. A trust acknowledgment ensures you maintain control over which third-party servers your agents can access.

    MCP Servers flip the script: instead of consuming external tools, you expose your Dimedove agents as tools that other applications can use. Create a hosted MCP server from your dashboard, assign an agent, select which tasks to expose, and optionally enable knowledge base access. Dimedove manages all the infrastructure, so there are no servers to deploy or maintain. Your agents become accessible to ChatGPT apps, AI automation tools like n8n, Make, Gumloop, and Zapier, AI assistants and copilots, and any MCP-compatible client. Built-in rate limiting and analytics give you visibility and control over external usage.

    - **MCP Tasks** for connecting agents to external MCP servers and third-party tools
    - **MCP Servers** for exposing your agents via Dimedove-hosted MCP endpoints
    - **Selective task exposure** to control exactly what external tools can access
    - **Knowledge base access** toggle for MCP server queries
    - **Connection testing** to verify MCP configurations before going live
    - **Built-in rate limiting** and usage analytics for MCP servers

    ### Features

    - Create MCP tasks that connect to any external MCP server during conversations
    - Configure server URL, transport protocol (Streamable HTTP or SSE), and custom headers
    - Automatic tool discovery from connected MCP servers
    - Trust acknowledgment workflow for third-party MCP server connections
    - Create and manage MCP servers from Settings > MCP in your dashboard
    - Assign any published agent to an MCP server
    - Selectively expose specific agent tasks through each MCP server
    - Enable or disable knowledge base queries per MCP server
    - Test MCP server connections and view discovered tools before enabling
    - Copy MCP server URLs for easy configuration in external tools
    - Live/Draft status with validation ensuring proper configuration before enabling
    - Per-server rate limiting to prevent abuse and ensure reliable performance
    - All MCP tool calls logged for monitoring and debugging

    ### Notes

    - MCP Tasks and MCP Servers are available to teams on the Pro subscription

</Update>
